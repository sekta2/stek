# 📚 Документация >> 🍱 Рецепты
Система рецептов, используется для крафтов в тулбоксе, воркбенче и прочее.

## 🛠️ Функции

### stek_recipes.add_recipe(индетификатор, данные)
Добавляет новый рецепт

- индетификатор - строка(string) - wood_table
- данные - таблица(table)
    - name - название
    - desc - описание
    - craftables - ресурсы для крафта - таблица где ключ является числовым айди ресурса а значение - количество ресурса
    - craft_type - где используется этот рецепт, toolbox или workbench

    - category - категория, например Props

    - model - моделька
    - hull - OBB модельки

    - func - функция спавна, в аргументы даётся игрок и позиция крафта

### stek_recipes.get_recipe(индетификатор)
Возвращает рецепт(таблицу) по строковому индетификатору

- индетификатор - строка(string) - wood_table

### stek_recipes.get_recipe_by_uid(индетификатор)
Возвращает рецепт(таблицу) по числовому индетификатору

- индетификатор - число(number) - 1 или в коде можно так - STEK_RECIPE_WOOD_TABLE

### stek_recipes.get_recipes(тип-крафта: НЕ ОБЯЗАТЕЛЬНО)
Возвращает все рецепты или по типу крафта, например toolbox или workbench

- тип-крафта - строка(string) - toolbox

### stek_recipes.add_prop_recipe(индетификатор, моделька, название, описание, категория, тип-крафта, ресурсы, хулл)
Вспомогательная функция которая добавляет в рецепты проп

- индетификатор - строка(string) - wood_table
- моделька - строка(string)
- название - строка(string)
- описание - строка(string)
- категория - строка(string)
- тип-крафта - строка(string)
- ресурсы - таблица(table)
- хулл - таблица(table)

### stek_recipes.init()
Системная функция которая создаёт все рецепты.

## ❓ Куда добавлять рецепты
В функцию `stek_recipes.init()` в самом низу скрипта `lua/stek/sh/recipes.lua`
